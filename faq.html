Installing the Mod

This mod requires Minecraft Forge. The minimum required build is listed next to each build in the Download section.

To install on a Windows client machine, place the mod zip file into your Minecraft directory under the mod folder, typically located at: %APPDATA%/.minecraft/mods/

To install on a server, place the mod zip file into the server mod folder.

If you're having difficulty installing the mod, avi12 has provided video installation instructions.

Migrating 1.6.X worlds to 1.7.X

To migrate your Carpenter's Blocks 1.6.4 worlds to 1.7.2, follow these steps:

  1. Load Carpenter's Blocks v2.1.2 or later for MC 1.6+ into your mods folder in the 1.6.4 subdirectory.
  2. Load Carpenter's Blocks v3.2.0 or later for MC 1.7+ into your mods folder in the 1.7.2 subdirectory.
  3. Start the game in Minecraft 1.6.4 and load the world.
  4. Ensure ALL chunks with Carpenter's Blocks have been loaded into memory by visiting all locations where the blocks are used.
  5. Exit game.

The blocks should now be prepared for migration. Start the game in Minecraft 1.7.2 and load the world. If the blocks don't migrate properly, try using a later version of Forge (build 1121 is recommended).

What won't survive the migration:

  • Multi-block entities like placed doors and beds may disappear, so it's recommended that you remove them first.
  • Chisel patterns and designs will disappear from placed blocks.

My game crashed! What do I do?

You can either post it in spoilers in the official thread if not already reported (do your best to search first), or as a new issue here if it doesn't already exist.

Chisel designs, fancy water, ice, colored glass.. none of it renders correctly in 1.7.2 and up, why?

After Minecraft 1.6.4, Mojang introduced new alpha rendering code that works great for ordinary blocks, but has proven to be troublesome for mods like this one, causing it to throw PriorityQueue exceptions and crash the game. Therefore, alpha rendering is currently disabled and the aforementioned features (chisel designs, covers, etc) may not look the way you expect. This is not a bug, and I am currently looking for ways to resolve it.

Will you include a localization file for my region?

Submit a pull request to the Github repository in the navigation bar at the top to have your language file(s) added.

To load your own language files right away, simply drop the .lang file into the zip in the folder assets/carpentersblocks/lang/ and they will be loaded at runtime.

Why won't game NPCs detect barriers, doors, etc? And why won't leads tie to Carpenter's Barrier posts?

Many aspects of Minecraft are hard-coded to certain objects, like horse leads only surviving when tied to vanilla fence posts, and NPCs only detecting vanilla doors. There are ways around this, but it's currently a low priority.

I'm having issues with this mod and MCPatcher, why?

This bug affected older builds primarily for Minecraft version 1.6. The latest builds in the 3.0.0 range are no longer affected.

Power isn't propagating through the Carpenter's Block. Why not?

The method to enable this functionality doesn't take into account any of the block's unique properties. Because of this, this feature is disabled. If forced, block shading is also effected negatively.
